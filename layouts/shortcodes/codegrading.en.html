Lab grading: code style

ATTENTION, this laboratory is graded in a different way, coding clarity and correctness is taken into account. Your codes will be checked after the class, see details below.

Points distribution: 8 for entry test, 16 for the graded task and up to 10 negative points for coding correctness and style flaws. What results in negative points:

  • not checking for system and library functions errors
  • not released resources
  • functions longer than 40 lines
  • unjustified global variables
  • static numbers within the code that should be constants or variables
  • use of unsafe or obsolete functions like sprintf, gets, scanf with %s format (buffer overflow)
  • additionally if the program handles signals:
    • lack of functions interrupted before they start restarting (EINTR)
    • lack of function interrupted in the middle of processing restarting (sleep, nanosleep, read. write)
    • program logic placed inside of signal handler, especially if this logic blocks the main code of the process for longer than necessary
